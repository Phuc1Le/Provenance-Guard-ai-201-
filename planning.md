# System design: Provenance Guard

## Simple Pipeline
```
User submits text-based content (e.g. a poem, a short story excerpt, or a blog post) for attribution analysis through the /submit endpoint. The user's rate limit is checked first. If the user has not exceeded the limit, the content is analyzed using two independent detection signals: an LLM-based semantic classifier and stylometric heuristics. The two signals are combined into a single AI-likelihood confidence score (0–1). Based on predefined thresholds, the confidence score is mapped to a reader-facing transparency label. The request, prediction, confidence score, and supporting evidence are logged into the database.

If the user finds the result unsatisfactory (for instance, their self-written work is classified as AI-generated), they may submit an appeal through the /appeal endpoint. The user's rate limit is checked again. If the user has not exceeded the limit, the server responds with an "Under Review" message. No automated review is performed. The appeal is logged into the database, and the original submission's status is updated to "Under Review".
```

## Classification model

### Confidence score

The classifier produces a confidence score between **0** and **1**, representing the estimated likelihood that the submitted content was generated by AI.

- **0.0** indicates strong evidence that the content was written by a human.
- **1.0** indicates strong evidence that the content was generated by AI.

The confidence score is mapped to one of five transparency labels:

- **Definitely Human**
- **Likely Human**
- **Uncertain**
- **Likely AI**
- **Definitely AI**

The thresholds are intentionally asymmetric to reduce false positives (incorrectly classifying human-written content as AI-generated), which are considered more harmful than false negatives.

### Edge cases

The system will typically return an **Uncertain** label when:

- The LLM and stylometric signals disagree significantly.
- The submitted text is too short to provide reliable stylistic evidence.
- The writing style naturally resembles AI output (e.g. technical documentation, research papers, or highly formal writing).
- AI-generated text has been substantially rewritten by a human, or human-written text has been heavily assisted by AI tools.

## Classification workflow

User submits content to the **/submit** endpoint. The user's request count is updated and checked against the rate limit. If the limit is exceeded, the server responds with **"Limit reached"**.

Otherwise, the content is analyzed using two independent detection signals:

1. **LLM-based semantic classification**, which evaluates the overall writing style and returns an AI-likelihood score together with structured reasoning.
2. **Stylometric heuristics**, which compute structural writing statistics (e.g. sentence length variation, vocabulary diversity, punctuation density, and sentence complexity) and return an independent AI-likelihood score.

The two scores are combined into a final AI-likelihood confidence score. Based on predefined thresholds, this score is mapped to one of the three transparency labels. The server returns:

- the transparency label,
- the confidence score,
- a concise explanation generated from the LLM's reasoning.

The prediction, confidence score, reasoning, and supporting evidence are then logged into the database, under the "classification" table.
## Appeal workflow

Users may dispute a classification by submitting an appeal.

### POST /appeal

Request:

```json
{
    "user_id": "",
    "post_id": "",
    "reason": ""
}
```

Workflow:

1. Validate that the post exists.
2. If the post status is already `"under_review"`, return a message indicating that an appeal has already been submitted.
3. Otherwise:
   - Insert a new record into the `appeals` table.
   - Update the corresponding row in the `classifications` table by changing its status from `"completed"` to `"under_review"`.
4. Return a confirmation message.

Appeals do **not** trigger automated re-classification. They are logged for transparency and future manual review.

## Multi-signal detection pipeline

### 1. LLM semantic classification

Purpose:

Evaluate the semantic and stylistic characteristics of the writing.

System prompt:

```text
You are an AI content attribution assistant.

Your task is to estimate how likely the given text was generated by an AI language model.

Do not rely on a single cue. Consider writing style, coherence, consistency, vocabulary usage, repetition, transitions, and overall naturalness.

Respond ONLY with valid JSON in the following format:

{
    "score": 0.0,
    "reasoning": [
        "...",
        "...",
        "..."
    ]
}

Rules:
- score ranges from 0.0 to 1.0
- 0.0 means strong evidence of human writing
- 1.0 means strong evidence of AI generation
- reasoning should contain concise observations, not chain-of-thought
```

User prompt

```text
Evaluate the following text.

{text}
```

---

### 2. Stylometric heuristics

Compute the following structural features.

#### Sentence length variation

Compute the standard deviation of sentence lengths.

- Low variation → contributes toward AI
- High variation → contributes toward human

---

#### Type-token ratio

Only compute if the submission contains at least **150 words**.

```
unique_words / total_words
```

- Low ratio → contributes toward AI
- High ratio → contributes toward human

---

#### Paragraph length variation

Only compute if there are at least **3 paragraphs**.

Compute the variance of paragraph lengths.

- Low variation → contributes toward AI
- High variation → contributes toward human

---

#### Longest sentence length

Compute the number of words in the longest sentence.

Suggested thresholds:

- >40 words → slight evidence toward human
- <20 words → slight evidence toward AI

---

The heuristic scores are normalized and combined into a single stylometric score between **0** and **1**.

---

## Confidence score interpretation

The LLM score and stylometric score are combined using a weighted average.

Suggested weights:

- LLM semantic score: **70%**
- Stylometric score: **30%**

The final score represents the estimated likelihood that the content was AI-generated.

Recommended thresholds:

| AI-likelihood score | Transparency label |
|---------------------|--------------------|
| 0.00 – 0.15 | Definitely Human |
| 0.15 – 0.35 | Likely Human |
| 0.35 – 0.65 | Uncertain |
| 0.65 – 0.90 | Likely AI |
| 0.90 – 1.00 | Definitely AI |

### Threshold rationale

These thresholds are chosen based on the project's goal of minimizing false positives.

- Scores below **0.15** indicate both detection signals strongly suggest human authorship, allowing the system to confidently classify the content as **Definitely Human**.
- Scores between **0.15** and **0.35** lean toward human authorship but still contain some AI-like characteristics, resulting in **Likely Human**.
- Scores between **0.35** and **0.65** indicate mixed or conflicting evidence. Rather than forcing a decision, the system returns **Uncertain**.
- Scores between **0.65** and **0.90** suggest the content is likely AI-generated, but not with overwhelming certainty. Since false positives are considered more harmful than false negatives, the system remains cautious before making a definitive AI attribution.
- Only scores above **0.90**, where both semantic and stylometric signals strongly agree, are classified as **Definitely AI**.

This design intentionally requires stronger evidence to make a definitive AI attribution than to express uncertainty, reducing the chance of incorrectly labelling genuine human writing as AI-generated.

## Displayed labels

### Definitely Human

Possible messages:

- Our analysis found strong evidence that this content was written by a human.
- This content exhibits characteristics strongly associated with human writing.
- We found little evidence suggesting this content was AI-generated.
- Based on our analysis, this content is almost certainly human-written.

---

### Likely Human

Possible messages:

- This content is likely to have been written by a human.
- Most of the evidence suggests human authorship.
- The writing appears predominantly human, although some AI-like characteristics were detected.
- Our analysis leans toward this content being human-written.

---

### Uncertain

Possible messages:

- Our analysis found mixed signals and could not confidently determine the origin of this content.
- This content contains characteristics of both human and AI writing.
- The available evidence is inconclusive.
- We cannot confidently determine whether this content was written by a person or generated by AI.

---

### Likely AI

Possible messages:

- This content is likely to have been generated by AI.
- Most of the evidence suggests AI-generated writing.
- The writing exhibits several characteristics commonly associated with AI-generated text.
- Our analysis leans toward this content being AI-generated.

---

### Definitely AI

Possible messages:

- Our analysis found strong evidence that this content was generated by AI.
- This content appears to be AI-generated with high confidence.
- The writing exhibits multiple characteristics strongly associated with AI-generated text.
- Based on our analysis, this content is almost certainly AI-generated.

## Guardrails

### Input length

- Fewer than **50 words** → return:

> Unable to classify: content is too short to provide reliable evidence.

- More than **3,000 words** → return:

> Content is too long. Please split the document into smaller sections.

---

### Prompt injection

Before sending text to the LLM, detect common prompt-injection attempts, such as:

- "Ignore previous instructions"
- "You are..."
- "Act as..."
- "Pretend..."
- "Suppose this was written by..."

These phrases should be treated as part of the submitted content rather than instructions to the model.

---

# Rate limiting

Each user may submit **5 classification requests**.

The request count is stored in the `users` table and enforced using Flask-Limiter.

Appeals do not count toward this limit.

---

# Audit log

Every classification request is recorded in the database, including:

- input text
- AI-likelihood score
- transparency label
- LLM reasoning
- stylometric score
- timestamp
- appeal status
